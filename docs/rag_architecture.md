# RAG Architecture (Retrieval Augmented Generation)

This document explains how the RAG pipeline works within the "Gemelo Cognitivo" project. The goal of this system is to retrieve the most relevant courses from our catalog based on the user's competency gaps.

## 1. Concept: What is RAG?
**Simple Version:**
Imagine a librarian who doesn't just look for keywords in titles but understands the *meaning* of the books. When you say "I want to learn how to store data", the librarian brings you books on SQL, NoSQL, and Databases, even if they don't have the word "store" in the title.

**Technical Version:**
RAG combines semantic search with LLM generation. Instead of training the model on our data, we retrieve relevant documents from a **Vector Database (ChromaDB)** and inject them into the LLM's prompt. This reduces hallucinations and ensures the roadmap is based on our actual course catalog.

---

## 2. Phase A: Indexing (Storage)
File: `agents/rag/indexer.py`

1.  **Extraction:** We read the `courses.json` file.
2.  **Structuring:** For each course, we create a descriptive text string combining title, description, competencies, and level.
3.  **Embedding:** We use a local model (`all-MiniLM-L6-v2`) to transform that text into a **384-dimensional vector** (a list of numbers representing meaning).
4.  **Persistence:** The vectors and their metadata are stored in **ChromaDB**.

## 3. Phase B: Retrieval (Search)
File: `agents/rag/retriever.py`

1.  **Gap Analysis:** The system reads the `competency_profile` generated by the Profiling Agent.
2.  **Query Construction:** It identifies competencies with a "low" level and builds a semantic search query (e.g., *"Courses focusing on: Python, SQL"*).
3.  **Vector Search:** It converts the query into a vector and calculates the **Cosine Similarity** against the courses in ChromaDB.
4.  **Top-K Results:** It retrieves the top 5 most relevant courses (filtering by metadata like level) to be used by the Planning Agent.

---

## 4. Key Technologies
- **Vector DB:** ChromaDB (Local & Persistent).
- **Embedding Model:** `all-MiniLM-L6-v2` (Sentence Transformers).
- **Metric:** Cosine Similarity (measures the angle between vectors to find semantic proximity).
